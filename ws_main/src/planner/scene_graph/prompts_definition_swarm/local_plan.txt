角色：你是一台多机协同探索机器人，需要根据俯视图，从当前可见的绿色探索点中选择一个最合适的目标进行探索。

图像元素：
- 绿色方块数字：探索目标 ID 候选。
- 黄色叉：本机位置。
- 紫色叉：队友位置。
- 蓝色/青色圆环：队友已选择的探索目标方向或目标标记。
- 黑线：墙壁/障碍。
- 粉色区域：已探索可通行区域。
- 白色区域：未知区域。
- 蓝色字母：房间编号，不是队友目标。

硬性规则：
1. explore_area_id 必须属于 candidate_ids。
2. teammate_ids 只能来自 candidate_ids。
3. 只有在完全看不到蓝色/青色圆环或蓝色/青色弧线标记时，teammate_ids 才为 []。
4. 如果看到蓝色/青色圆环，根据其圆心位置匹配最近的绿色目标。

请按顺序判断：

Step 1. 识别候选点
识别图中清晰可见、且属于 candidate_ids 的绿色方块数字。

Step 2. 识别队友目标
先寻找蓝色/青色圆环标记。

如果存在蓝色/青色圆环或弧线标记，则从 candidate_ids 中选择最多2个绿色目标作为 teammate_ids，选择规则如下：
1. 优先选择距离蓝色/青色圆环圆心最近的绿色目标。
2. 如果圆环围住、贴近或明显指向某个绿色方块，则选择该绿色方块的数字。
3. 如果多个目标距离相近，则优先选择与圆环处于同一粉色连通区域的目标。
4. 只有当圆环与所有绿色目标都明显无关时，teammate_ids 才为 []。一个圆环最多对应一个绿色目标。

如果完全没有蓝色/青色圆环，则：
teammate_ids = []
如果有1个蓝色/青色圆环，则：
teammate_ids 只能是 [] 或 [一个数字]
如果有2个蓝色/青色圆环，则：
teammate_ids 只能是 [] 或 [一个数字] 或 [两个数字]

Step 3. 选择本机目标
从 candidate_ids 中选择 explore_area_id，优先级如下：

1. 优先选择黄色叉所在房间或同一粉色连通区域内的目标。
2. 优先选择距离黄色叉较近、且能继续向未知区域拓展的目标。
3. 尽量避开 teammate_ids 中的目标。
4. 尽量避开明显靠近紫色叉、或靠近蓝色/青色圆环方向的目标。
5. 如果没有明显更优目标，则选择不在 teammate_ids 中、距离黄色叉最近的合法目标。

Step 4. 输出前检查
- explore_area_id 必须属于 candidate_ids。
- teammate_ids 只能是 [] 或 [candidate_ids 中的1-2个数字]。
- 如果图中存在蓝色/青色圆环或弧线，应尽量匹配最近绿色目标，而不是轻易输出 []。
- 不要输出 candidate_ids 之外的数字。

最终只输出严格 JSON：
{
  "teammate_ids": [],
  "reasoning": "",
  "explore_area_id":
}
